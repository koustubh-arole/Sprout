Sprout — Status & Product Roadmap

From hackathon MVP to a robust, research-grounded product

Prompt Wars Virtual Challenge 3 · "Create behaviour, not just awareness." · Internal briefing · 17 June 2026

1 · Executive summary

Sprout has moved well beyond the original "living world + nudge" MVP. It is now a virtual-companion (creature) carbon platform: you adopt and name a creature, perform one verified high-impact green action a day, and watch the creature grow, level up, and thrive — backed by streaks, collectibles, commitment pacts, a social grove (leaderboard), sponsor challenges, a community blog, and an AI coach + chat. The build is a client-first Next.js web app with a clean data seam that already anticipates a real backend.

2.3 Information architecture

Five tabs: World (companion + daily ritual + coach), Grove (leaderboard), Pacts (challenges), Blog, You (profile). The decision-moment + footprint simulator is intentionally demoted to a "Curious? Simulate a choice" sandbox so the companion is the hero — the felt, emotional experience the brief rewards.

4 · Honest gaps, risks & what could go wrong

Risk Mock social layer reads as "fake"

Leaderboard, friends, challenges and profile rank are static mock data. A sharp judge or real user will notice there are no real people. Credibility hit if presented as live.

Risk Mock verification = integrity gap

"Verified" green actions are a plausibility stub, not real photo AI. The whole "verified behaviour" promise rests on this. Must be clearly labelled until Phase C.

Risk No accounts / single-device

All state is localStorage. There is no auth, no cross-device sync, and a "leaderboard" cannot be genuinely social without real users.

Risk Client-clock streaks are spoofable

Streak/daily-care logic trusts the device clock; trivially gamed. Needs server-side time + anti-cheat for any real competition.

Gap Photos as data-URLs in localStorage

Bloats storage, no real upload/storage, lost on clear. Needs object storage.

Gap Test depth

Pure logic is tested; the new adopt → ritual → reward flows have no component/e2e coverage.

Gap Hollow-reward trap

Collectibles/points don't yet connect to real-world impact (the failure the research flagged: "points that lead nowhere").

Gap Retention mechanics

No recurring weekly feedback or "you vs your past self" — both strongly evidence-backed against novelty decay.

6 · MVP → Product roadmap

Phase A — Harden & ship (submission-critical, days)

  • Run typecheck + lint + test:run + build clean; fix any failures.
  • Transparently label all mock data in-UI (a small "demo data" chip) so simulated social proof never reads as deceptive.
  • Add component/e2e smoke tests for adopt → daily ritual → reward → streak.
  • Deploy to Vercel; verify deployed link in incognito; confirm no API key in client bundle.
  • Write README (architecture + AI-tool + prompt-flow narrative) and the LinkedIn post.

Phase B — Real backend & accounts (the biggest leap)

  • Add Supabase auth + Postgres; implement lib/data/supabase.ts behind the existing SproutData seam — components don't change.
  • Real leaderboard / friends / pacts / profile rank; invite codes for hostel-floor & office-team groups (maps to dorm-competition evidence).
  • Server-side time & streak validation + rate-limits (anti-cheat) so competition is credible.
  • Photo upload to Supabase Storage; Row-Level Security on all tables.
  • Keep client-first fallback so the deployed link still degrades gracefully.

Phase C — Real verification & smarter AI

  • Replace mock verifyActivity with Gemini vision photo verification (the integrity core).
  • RAG-ground the coach on the curated India factor table to hard-stop hallucinated numbers.
  • Personalised weekly recap (the recurring-feedback lever) + the app's own AI-footprint disclosure (honesty hook).

Phase D — Retention, scale & impact

  • Recurring weekly feedback + "you vs your past self" trend (self-comparison ≈14% reductions, sometimes beats neighbour comparison).
  • Real-world impact reward: tie collectibles/points to a verified tree-planting partner so rewards aren't hollow (Ant Forest / Forest model; treat offsets as last-resort, not core).
  • Co-op play (co-raise a creature, help a friend's streak) — Ant Forest's sustained-engagement mechanic.
  • Accuracy upgrade: Climatiq/DEFRA-backed factors with disclosed vintage + uncertainty ranges; PWA + offline; Hindi/regional i18n; email/push nudges; retention analytics; B2B campus/office + verified-impact sponsor challenges (monetization).

Open research questions to validate (post-launch)

  • Does the companion/pet framing retain Indian hostel/office users better than a pure living-world? (A/B)
  • Optimal cadence of recurring feedback before it becomes noise.
  • The real high-impact action set for growth-stage users (first AC, first car, first flights).
  • A verification UX that adds integrity without re-introducing tracking fatigue.

8 · Recommended next steps (in order)

  1. Ship Phase A: green build, label mocks, smoke tests, deploy, README + LinkedIn. (unblocks submission)
  2. Supabase adapter (Phase B) behind the data seam — real accounts + real social. (largest product jump)
  3. Self-comparison + weekly recap — cheap, high-retention.
  4. Gemini-vision verification (Phase C) — turns "verified" from a promise into a fact.
  5. RAG-grounded coach + AI-footprint disclosure — accuracy + credibility.
  6. Real-impact rewards — fixes the hollow-reward failure mode.
One rule to protect the score: the latest submission is the one that counts. Treat the final attempt as sacred — only re-submit after the deployed link is verified working and the build is green. Every new "product" feature should ship behind the data seam and keep the client-first fallback intact, so the judged link can never break.

9 · Verification & QA checklist

  • npm run typecheck · npm run lint · npm run test:run · npm run build — all clean.
  • Manual: adopt → daily ritual (with photo) → creature heals → reward opens → streak increments.
  • Coach: with key (live AI) and with key unset (fallback still returns sensible coaching).
  • A11y: keyboard-only traverse each tab; screen-reader announces world/creature state; reduced-motion honoured.
  • Security: grep built client bundle for the API key (must be absent); .env ignored, .env.example committed.
  • Deployed link opens cleanly in incognito with no console errors.
